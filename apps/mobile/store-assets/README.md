# Senzilytics mobile store assets

Store requirements change. Confirm current dimensions and device classes in App Store Connect and Google Play Console before capture.

- `google-play-feature-graphic.svg` — editable 1024×500 source
- `google-play-feature-graphic.png` — upload-ready 1024×500 render
- `../assets/app-icon.png` — native 1024×1024 source icon
- `../../../public/icons/senzilytics-512.png` — Google Play 512×512 icon from the same source

## Screenshot story

Capture real application screens from a dedicated fictional Premium review tenant. Do not use production tenant data, employee names, incident details, email addresses or customer identifiers.

Recommended sequence:

1. **Your field command center** — Home with fictional tasks and workspace metrics.
2. **Capture observations anywhere** — completed observation form with fictional content.
3. **Work through unreliable connectivity** — offline-ready state or queued record.
4. **Stay ahead of assigned work** — fictional alerts and assigned workflow.
5. **Security follows the user** — Settings showing tenant identity, data protection and account controls.

Capture at least one supported phone class for both stores and an iPad/tablet class because the app supports tablets. Screenshots should show the app in use rather than only the login or splash screen.

## Visual rules

- Keep the Senzilytics cyan S mark, navy background and approved typography consistent.
- Use short factual captions; avoid unverifiable rankings, certification claims or guarantees.
- Do not add Apple, Android, Microsoft or Okta logos to screenshots or the feature graphic.
- Preserve safe margins so store cropping does not cut the mark or text.
- Review every image at actual store-preview size before upload.
